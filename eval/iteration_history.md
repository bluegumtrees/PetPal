## R1 → R3：+33 个百分点的来源

**基础设施桶（约 40 次失败）**：
1. **DeepInfra Qwen3 模板的角色规则**——对话以 assistant 消息结尾（含尾部 system 纠偏）
   → 立即 EOS 空响应。planner 所有兜底重试恰好都是这个形状，"需要抢救的轮次必死"。
   修复：7 处纠偏注入全部改 user 角色（`54f8b20`），本地 4/4 复现 → 4/4 治愈。
2. 本地→OpenRouter 时段性超时 + openai client 默认 600s 超时陪跑僵死请求。
   修复：timeout=90s + SDK 重试 2 次。

**行为桶（R3 前修复）**：
- 营养"常识题"跳过 RAG → prompt 明确营养/喂食量/禁忌食物必查（新增营养库 216 条后必要）
- "上次疫苗是什么时候"不调 query_pet_history → prompt 补例外条款
- 评测污染 fixture（评测自己 save 的事件累积）→ 每轮跑前 seed 重置

## R3 → R4：数字持平，质量提升

R3 的 3 个稳定失败全部治愈，代价是方差换了 3 个新面孔（见下）。结构性指标进步：
- 路由 89%→94%：视觉任务（emotion/bcs/pain_fgs）无图时降级 chat——纯文字"分离焦虑"
  不再被派给看图任务（a05 治愈）
- forbidden 84%→92%：`find_nearby_clinic` 位置护栏——主人没给位置时工具拒绝执行并
  引导反问，不再编造"北京海淀中关村"（a09 治愈）
- 延迟 41s→24s：纠偏不再空转整个 iter 预算
